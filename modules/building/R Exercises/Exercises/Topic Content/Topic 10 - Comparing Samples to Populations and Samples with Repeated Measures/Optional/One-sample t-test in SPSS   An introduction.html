Introduction

This tutorial will guide you through the process of running a one-sample t-test in SPSS, and interpreting and reporting the results. It will show you how to test all the assumptions that underlie this test and what to do when these assumptions are violated. You will also be shown how to report confidence intervals.

SPSStop ^

Assumptions & order of testing

For an one-sample t-test to be able to provide a valid result, the following three assumptions must hold:

  1. There are no outliers.
  2. The dependent variable is normally distributed in the population.
  3. The data are independent (i.e., not correlated/related).

These assumptions need to be tested before you can run a one-sample t-test. If these assumptions are not violated, the results from the one-sample t-test will be valid. However, if they are violated then you will need to either:

  1. Make corrections to the data, such as removing outliers and/or transforming data.
  2. Run a different type of statistical test.

You will be shown how to test for both these assumptions in this guide in the order presented above: outliers first and then normality. Depending on the presence of outliers or the normality of the data, you will be instructed on how to proceed with testing. This could include making corrections to your data or using an alternative statistical test. This will also include the route to take when, and if, any corrections you make still results in the data failing the assumptions of this test.

These choices will be clearly signposted for you as you work through the guide.

The first two assumptions will be tested using the Explore... procedure, whilst the one-sample t-test will be run using the One-Sample T Test procedure.
